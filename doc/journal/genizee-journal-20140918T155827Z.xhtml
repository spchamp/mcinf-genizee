A Formatting Pipeline for Garnet Documentation

The Garnet documentation is available, upstream, in its original TeX format, as well as in the PostScript presentaton format.

... PostScript to PDF ...

#!/bin/sh

FILES=*.ps

for PS in ${FILES}; do
        NAME=$(echo ${PS} | cut -d. -f1)
        PDF=${NAME}.pdf
        echo "Converting ${PS} to ${PDF}"
        ps2pdf ${PS} ${PDF}
done

... PDF to DjVu — in a simple example, none utilizing any of the additional formatting control options of the pdf2djvu shell command,, such as special handling for image quality, tables of contents, and Page numbering ...

#!/bin/sh

FILES=*.pdf

for PDF in ${FILES}; do
        NAME=$(echo ${PDF} | cut -d. -f1)
        DJ=${NAME}.djv
        echo "Converting ${PDF} to ${DJ}"
        pdf2djvu -o ${DJ} ${PDF} 
done

... though — as thusly, and by default — using lossless compression in the DjVu converter, then the respective djv files all wind up being of a larger file size than the secondary pdf files.
